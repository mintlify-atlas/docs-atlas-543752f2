## Overview

The Reports module (`/reporte`) is the primary interface for MINSA inspectors and the financial transparency tool for admins and collectors. It reads directly from the local IndexedDB store, so **every report can be generated fully offline** without any server connection.

| Role | Access Level |
|---|---|
| `minsa` | Primary user — redirected here automatically on login, read-only |
| `cobrador` | Secondary — can view and export financial summaries |
| `admin` | Secondary — full access to reports and export |

---

## Route

```
/reporte
```

The `ROLE_HOME` constant in `src/utils/constants.js` maps the `minsa` role directly to this route:

```js
export const ROLE_HOME = {
  admin:    '/admin',
  cobrador: '/cobros',
  minsa:    '/reporte',   // Inspector lands here automatically
  cliente:  '/historial',
  dev:      '/admin',
};
```

## Dashboard Metrics

Before any export, the page surfaces four live summary cards populated via Dexie `useLiveQuery` hooks:

| Card | Calculation |
|---|---|
| **Ingresos (Pagos)** | Sum of all `pagos.monto` values |
| **Egresos (Gastos)** | Sum of all `gastos.monto` values |
| **Balance Neto** | `totalIngresos − totalEgresos` |
| **Comunidad** | Count of all entries in `db.usuarios` |

All values are formatted in Panamanian balboa (B/.) using `formatMonto()` from `src/utils/formatters.js`.

## Filter Options

The report view supports filtering the dataset before export:

| Filter | Control | Description |
|---|---|---|
| **Desde** | Date picker | Start of date range (inclusive) |
| **Hasta** | Date picker | End of date range (inclusive) |
| **Ingresos** | Checkbox | Include `pagos` records in output |
| **Egresos** | Checkbox | Include `gastos` records in output |
| **Jornales** | Checkbox | Include attendance and fine records |

After selecting the desired filters, click **Aplicar Filtros** to update the preview table in real time.

## Report Types: What Gets Included

### Ingresos
All `pagos` records matching the date range, drawn from `db.pagos`:

| Field | Source | Description |
|---|---|---|
| `fecha` | `pagos.fecha` | Date of payment |
| `tipo` | `pagos.tipo` | Payment type (e.g., `mensual`, `parcial`, `multi_mes`) |
| `monto` | `pagos.monto` | Amount paid in B/. |
| `cobrador` | `pagos.cobrador` | Collector who registered the payment |

### Egresos
All `gastos` records matching the date range, drawn from `db.gastos`:

| Field | Source | Description |
|---|---|---|
| `fecha` | `gastos.fecha` | Date of expense |
| `descripcion` | `gastos.descripcion` | Description of the expense |
| `monto` | `gastos.monto` | Amount in B/. |

### Jornales
Community work attendance records from `db.jornales`, including:

- Member name and attendance status (`asistió`, `sustituto`, `faltó`)
- Substitute name (if applicable)
- Hours logged
- Fine amount (`multa`) for absences, defaulting to the configured `multaJornal` value (e.g., B/.10.00)

## The 8-Sheet Excel Report

The full export generated by `src/services/reportesService.js` produces an `.xlsx` workbook with **eight sheets**, providing a complete financial picture for MINSA auditors:

| # | Sheet Name | Contents |
|---|---|---|
| 1 | **Ingresos** | All payment records with date, type, and amount |
| 2 | **Egresos** | All expense records with date, description, and amount |
| 3 | **Jornales** | Attendance records with substitutes, hours, and fines |
| 4 | **Resumen** | Aggregated totals: total income, total expenses, net balance |
| 5 | **Detalle Cobros** | Line-by-line breakdown of each collection |
| 6 | **Estado Cuentas** | Per-resident account status (`activo`, `moroso`, `corte`) |
| 7 | **Comisiones** | Collector commission log with dev/collector split breakdown |
| 8 | **Análisis IA** | AI-generated risk scores, morosidad predictions, and anomaly flags |

Column widths are pre-configured for readability (e.g., 15 chars for dates, 40 chars for descriptions).

## Export: Excel (.xlsx)

The Excel export uses **SheetJS (`xlsx`)**, bundled locally so it works without internet.

```js
import * as XLSX from 'xlsx';

const wb = XLSX.utils.book_new();

const ingresosData = pagos.map(p => ({
  Fecha:   formatFecha(p.fecha),
  Detalle: `Pago ${p.tipo || 'Cuota'}`,
  Monto:   parseFloat(p.monto) || 3
}));

const wsIngresos = XLSX.utils.json_to_sheet(ingresosData);
wsIngresos['!cols'] = [{ wch: 15 }, { wch: 30 }, { wch: 15 }];
XLSX.utils.book_append_sheet(wb, wsIngresos, "Ingresos");

XLSX.writeFile(wb, `Reporte_SIMAP_${new Date().toISOString().split('T')[0]}.xlsx`);
```

**Filename format:** `Reporte_SIMAP_YYYY-MM-DD.xlsx`

## Export: PDF

The PDF export uses **jsPDF** + **jsPDF-autotable**, also bundled locally.

```js
import jsPDF from 'jspdf';
import autoTable from 'jspdf-autotable';

const doc = new jsPDF();

doc.setFontSize(18);
doc.text("Junta Administradora de Acueducto Rural (SIMAP)", 14, 20);
doc.text(`Reporte Financiero Oficial`, 14, 28);
doc.text(`Fecha de Corte: ${new Date().toLocaleDateString()}`, 14, 34);

autoTable(doc, {
  startY: 50,
  head: [['Fecha', 'Detalle', 'Monto']],
  body: ingresosBody,
  theme: 'grid',
  styles: { fontSize: 10 }
});

// Signature line
doc.text("_________________________", 14, finalY + 60);
doc.text("Firma del Presidente / Tesorero", 14, finalY + 66);

doc.save(`Reporte_SIMAP_${new Date().toISOString().split('T')[0]}.pdf`);
```

The PDF includes:
- Institution header with report date
- Ingresos table with `grid` theme
- Egresos table appended below
- Summary totals (Total Ingresos, Total Egresos, **Saldo a Favor** in bold)
- Signature line for the President / Treasurer

## MINSA Inspector Workflow

The following scenario matches **Escenario 4** from the project requirements (`docs/requisitos.md`):

<Steps>
  <Step title="Log in as minsa">
    Use credentials `minsa / 1234`. The system authenticates via `authService.js` and reads the `ROLE_HOME` mapping.
  </Step>
  <Step title="Automatic redirect to /reporte">
    The router guard detects the `minsa` role and redirects immediately to `/reporte` — no navigation required.
  </Step>
  <Step title="Set the date range">
    Enter the desired **Desde** (start) and **Hasta** (end) dates using the date pickers. For example: `Desde: 2026-01-01` → `Hasta: 2026-03-31` for a quarterly report.
  </Step>
  <Step title="Select report types">
    Check the boxes for **Ingresos**, **Egresos**, and/or **Jornales** depending on what the audit requires. Unchecking a type removes it from the preview and export.
  </Step>
  <Step title="Click Aplicar Filtros">
    The summary table updates in real time via `useLiveQuery`. Total ingresos, egresos, and net balance recalculate instantly.
  </Step>
  <Step title="Download Excel or PDF">
    - Click **📊 Descargar Excel** to download the full 8-sheet workbook (`Reporte_SIMAP_2026-03-31.xlsx`).
    - Click **📄 Descargar PDF Oficial** to download a formatted PDF with signature line.
  </Step>
</Steps>

## Offline Report Generation

<Note>
Reports are generated entirely from **local IndexedDB data** via Dexie.js — no network request is made during export. A MINSA inspector can download a complete Excel or PDF report even when the device has no internet connection, as long as the data has been synced at least once beforehand. Both the SheetJS and jsPDF libraries are bundled locally in the app.
</Note>
